# LogiSquare 요구사항 명세서 — UI 구성 / 인터페이스 요구사항

## UI 구성

### 1. 로그인 화면

- 입력 : 아이디, 비밀번호
- 로그인 시 (관리자: admin, 작업자: 사원번호) 역할에 따라 관리자 홈 또는 작업자 홈으로 이동
- 로그인 실패 시 "아이디 또는 비밀번호를 확인하세요" 메시지 표시, 아이디·비밀번호 중 빈 칸이 있으면 로그인 버튼 비활성화

### 2. 메인 홈

- 좌측 고정 메뉴: 홈 / 물류 / 안전 / 근태 (작업자는 호출 승인 메뉴 추가)
- 진행 중 작업 수, 가용 작업자 수, 안전 위반 건수, 입고 대기 건수(관리자) / 대기 중인 호출(작업자) 를 숫자 카드 4개로 표시
- 카드 클릭 시 다른 화면으로 이동하지 않고, 카드 아래에 해당 항목의 상세 목록이 펼쳐짐 (재클릭 시 접힘, 동시에 하나만 열림)

### 3. 물류 관리

- 상단 탭으로 하위 화면 전환, 탭 구성은 역할별로 다름 (관리자: 입고 및 배치 / 배치 현황, 작업자: 배치 현황 / 출고)

**3-1. 입고 및 배치 (관리자 전용)**
- 품목명 + 수량 입력 → "배치 위치 확인" 버튼 → 수량 기준 ABC 등급과 권장 배치 위치 안내
- "저장" 클릭 시 배치 확정, "배치 완료" 토스트 메시지 표시
- 저장 후 "작업자 호출(입고 지시)" 버튼 → 확인창(예/아니오) → 가용 작업자에게 입고 호출 전송

**3-2. 배치 현황 (관리자·작업자 공통)**
- 창고 그리드(4행 × 8열)를 등급 색으로 표시한 지도, 좌측에 출입구 표기 (출입구에서 가까운 열일수록 고회전 등급)
- 검색창에 품목명 입력 시 재고 수량·위치를 안내하고 지도에서 해당 슬롯 하이라이트
- 전체 재고 목록 테이블 제공, 행 클릭 시 검색·하이라이트로 연결
- (관리자만) 위치 확인 후 "작업자 호출(출고 지시)" 버튼 → 확인창(예/아니오) → 가용 작업자에게 출고 호출 전송

**3-3. 출고 (작업자 전용)**
- 재고 목록에서 품목 선택 → 출고 수량 입력(재고 초과 입력 방지) → "출고 처리" 버튼
- 처리 시 재고 수량 차감, 전량 출고되면 해당 슬롯이 빈 칸으로 전환

### 4. 안전 관리

- 우측 상단에 실시간 위반 알림 팝업 생성(구역·유형·시각 표시), 5초 후 자동 소멸, 발생 시 경고음 재생
- 상단에 지금 발생 중인 위반을 카드로 고정 표시, "조치 완료 처리" 가능
- 하단에 유형별로 분리된 위반 로그 테이블(보호장비 미착용 / 위험구역 침입), 발생 시각·구역·캡처 이미지 표시, 구역 검색 지원
- 위반 유형은 아이콘 + 색상으로 구분(보호장비 미착용=주황, 위험구역 침입=빨강)
- "새 위반 시뮬레이션" 버튼으로 위반 발생 상황을 데모 가능

### 5. 근태 관리

- 이름 / 호출 수락 횟수 / 처리 작업 건수 / 안전 위반 횟수 / 출근 / 퇴근 순서의 표, 이름 가나다순 정렬 고정
- (관리자만) 호출 수락 횟수·처리 작업 건수·안전 위반 횟수 중 하나를 클릭하면 하단에 작업자별 막대그래프 표시
- 일 / 주 / 월 / 년 단위로 수치를 환산해 표시하는 기간 필터
- 관리자는 전체 작업자, 작업자는 로그인한 본인 행만 표시(그래프 영역은 미노출)

### 6. 호출 승인 (작업자 전용)

- 대기 중인 호출(품목명, 위치, 요청 시각, 입고/출고 구분 배지) → "승인" 버튼
- 승인된 호출은 진행 중인 작업으로 이동 → "처리완료" 버튼으로 작업 종료 입력
- 처리완료된 호출은 이력 목록으로 이동(읽기 전용)
- 상태 흐름: 대기 → 승인 → 완료

## 인터페이스 요구사항

### 1. 사용자 인터페이스

- 웹 브라우저 기반 반응형 UI 제공 (데스크톱 1280px 이상 기준)
- 로그인 계정의 역할(관리자/작업자)에 따라 메뉴·화면 구성을 다르게 렌더링
- 사용자 액션(저장, 호출 전송, 승인 등) 처리 결과를 토스트 메시지로 3초간 노출(성공/오류 톤 구분)
- 작업자 호출 등 확인이 필요한 동작은 예/아니오 확인창으로 재확인
- 창고 배치 등급(A/B/C)은 모든 화면에서 동일한 색상 규칙으로 표시(A=고회전/레드, B=중회전/앰버, C=저회전/그린)
- 안전 위반 발생 시 시각적 강조(색상)와 청각적 알림(경고음)을 함께 제공
- 재고·근태·위반 로그 등 목록형 데이터에 텍스트 검색·필터링 제공
- 미인증 사용자가 인증이 필요한 화면에 접근하면 로그인 화면으로 리다이렉트
- 역할 범위를 벗어난 화면(예: 작업자의 입고 화면 접근)은 접근을 제한하고 허용된 화면으로 리다이렉트

### 2. 하드웨어 인터페이스

> 현재 프로토타입은 실제 하드웨어와 연동하지 않고 버튼 클릭으로 상황을 시뮬레이션함. 아래는 정식 서비스 전환 시 필요한 요구사항.

- 구역별 CCTV/카메라 영상으로 보호장비 미착용·위험구역 침입을 감지 (현재: "새 위반 시뮬레이션" 버튼으로 대체)
- 위반 감지 시 캡처 이미지를 저장·조회
- 위치 인식 장치(비콘/RFID 등)로 작업자 위치를 파악해 "가장 가까운 작업자 호출" 로직에 활용 (현재: 가용 작업자 중 첫 번째를 배정)
- 출입 인증 장치(카드/생체 인증 등)로 출퇴근 이벤트를 수신 (현재: 목업 데이터로 대체)
- 데스크톱 브라우저 및 작업 현장에서 사용 가능한 태블릿 환경 지원

### 3. 소프트웨어 인터페이스

- React 19 + TypeScript 기반 클라이언트, Vite를 빌드 도구로 사용
- React Router 기반 클라이언트 사이드 라우팅으로 SPA 동작 (화면 전환 시 서버 재요청 없음)
- Tailwind CSS로 스타일 구성, 최신 2개 버전의 Chrome/Edge/Safari 지원
- 정식 서비스 전환 시 REST API 백엔드와 통신할 데이터 접근 계층 필요 (현재: React Context 메모리 상태로만 관리, 새로고침 시 초기화)
- 정식 서비스에서는 토큰 기반 인증(JWT 등)·보안 저장소 필요 (현재: 세션을 저장하지 않는 인메모리 로그인)

### 4. 통신 인터페이스

> 현재 프로토타입은 프런트엔드 단독으로 동작하며 외부 통신이 없음. 아래는 정식 서비스에서 요구되는 통신 인터페이스.

- 클라이언트-서버 간 통신에 HTTPS(TLS 암호화) 사용
- 안전 위반·작업자 호출 등 실시간 이벤트 수신 (WebSocket 또는 Server-Sent Events 권장, 현재는 클라이언트 내 로컬 상태 갱신으로 대체)
- 작업자 호출 발생 시 대상 작업자에게 푸시 알림 전송 (현재는 같은 브라우저 세션 내 상태 공유로 대체)
- CCTV/센서 등 현장 장비와 서버 간 데이터 수집을 위한 IoT 통신 프로토콜(MQTT 등) 지원

## 용어 정리

- **등급(A/B/C)**: 창고 슬롯의 회전율 구간 분류. A(고회전, 입구 인접) → B(중회전) → C(저회전, 안쪽)
- **배치 현황**: 창고 그리드 지도 상에서 어떤 품목이 어느 슬롯에 있는지 보여주는 화면
- **호출**: 관리자가 특정 작업(입고 배치 또는 출고 처리)을 수행하도록 작업자에게 보내는 업무 지시
- **처리완료**: 작업자가 승인한 호출 건의 실제 작업을 마쳤음을 시스템에 기록하는 행위 (재고 데이터 자체를 변경하지는 않음)
